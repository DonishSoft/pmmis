## 1. Общее описание модуля

Модуль **«Отчёты»** предоставляет аналитические инструменты для мониторинга ключевых показателей проекта WSIP. Модуль включает три независимых отчёта, каждый из которых доступен через боковое меню в разделе **«Отчётность»**.

### Состав модуля

| № | Отчёт | Назначение | Путь |
|---|-------|------------|------|
| 1 | **Отчёт по платежам** | Финансовый анализ контрактов | `/Reports/Payments` |
| 2 | **Отчёт по индикаторам** | Прогресс целевых показателей | `/Reports/Indicators` |
| 3 | **KPI сотрудников** | Эффективность персонала | `/Reports/EmployeeKpi` |

### Общие возможности

- 📊 **Интерактивные графики** (Chart.js) — столбчатые, круговые диаграммы
- 📋 **Детализированные таблицы** с раскрывающимися строками
- 🖨️ **Печать** — кнопка вывода на печать на каждой странице
- ⚡ **Асинхронная загрузка** — данные загружаются через AJAX с анимацией скелетона
- 🔐 **Раздельные права** — каждый отчёт управляется отдельным разрешением

### 2.1. Сводные карточки

В верхней части страницы отображаются 4 ключевых показателя:

| Карточка | Описание | Пример |

| **Запланировано** | Суммарная плановая стоимость всех контрактов (USD) | $17 948 939,46 |

| **Остаток** | Разница между планом и оплатой | $15 901 153,17 |

### 2.2. Графики

#### Столбчатая диаграмма «Планируемые vs Оплаченные»
- **Тип:** Сгруппированная столбчатая диаграмма (Bar Chart)
- **Ось X:** Номера контрактов
- **Ось Y:** Суммы в USD
- **Наборы данных:**
  - 🔵 Синий — запланированная сумма
  - 🟢 Зелёный — оплаченная сумма

#### Круговая диаграмма «Распределение оплат»
- **Тип:** Кольцевая диаграмма (Doughnut Chart)
- **Данные:** Доля оплаченных сумм по каждому контракту
- **Легенда:** Расположена снизу с названиями контрактов

### 2.3. Детализация по контрактам

Основная таблица содержит следующие столбцы:

| Столбец | Описание |
|---------|----------|
| **Контракт** | Номер контракта (ссылка на редактирование) и код проекта |
| **Подрядчик** | Наименование подрядчика |
| **Валюта** | USD 💵 или TJS 🇹🇯 с обменным курсом при подписании |
| **План** | Плановая сумма (в TJS + эквивалент USD для сомони-контрактов) |
| **Оплачено** | Фактически оплачено (в TJS + эквивалент USD) |
| **Курсовая разница** | ▲ прибыль / ▼ убыток от изменения курса USD/TJS |
| **Остаток** | Неоплаченная разница |
| **%** | Процент выполнения с прогресс-баром |

#### Раскрывающаяся детализация платежей

При клике на строку контракта раскрывается вложенная таблица с историей каждого платежа:

| Поле | Описание |
|------|----------|
| Дата | Дата совершения платежа |
| Тип | Аванс / Промежуточный / Окончательный / Удержание |
| Статус | ⏳ Ожидает / ✓ Одобрен / ✅ Оплачен / ❌ Отклонён |
| Сумма (TJS) | Сумма в сомони (для TJS-контрактов) |
| Курс оплаты | Курс НБТ на дату платежа |
| USD (по курсу контракта) | Сумма в USD по курсу на дату подписания |
| USD (по курсу оплаты) | Сумма в USD по курсу на дату оплаты |
| Курсовая разница | Разница между двумя USD-суммами |

### 2.4. Особенности расчёта курсовой разницы

Для контрактов в **TJS** (сомони) система автоматически рассчитывает:

```
USD по курсу контракта = Сумма TJS ÷ Курс при подписании
USD по курсу оплаты    = Сумма TJS ÷ Курс НБТ на дату платежа
Курсовая разница       = USD по курсу оплаты − USD по курсу контракта
```

- **▲ Положительная** (зелёный) — курс вырос, экономия в USD
- **▼ Отрицательная** (красный) — курс упал, дополнительные расходы

### 2.5. Итоговая строка

В подвале таблицы отображаются агрегированные показатели по **всем контрактам**:
- Общий план (USD)
- Общая оплата (USD)
- Суммарная курсовая разница
- Общий остаток
- Общий процент выполнения

## 3. Отчёт по индикаторам

**Путь:** `/Reports/Indicators`  
**Назначение:** Мониторинг прогресса достижения целевых показателей проекта WSIP

![Отчёт по индикаторам](/Users/mirqurbonnaimov/.gemini/antigravity/brain/6dcbc87c-3bb8-4347-9236-729e9a7d826d/indicators_report_1771843476085.png)

### 3.1. Сводные карточки

| Карточка | Описание |
|----------|----------|
| **Всего индикаторов** | Общее количество зарегистрированных индикаторов проекта |
| **Выполнено** | Количество индикаторов, достигших 100% и более |
| **В процессе** | Количество индикаторов с частичным прогрессом (0–99%) |
| **Общий прогресс** | Средний процент достижения по всем индикаторам с прогресс-баром |

### 3.2. График прогресса

- **Тип:** Сгруппированная столбчатая диаграмма (Bar Chart)
- **Ось X:** Коды индикаторов (IR-1, IR-1.1, PDO-1 и т.д.)
- **Ось Y:** Числовые значения
- **Наборы данных:**
  - 🔵 Синий — целевое значение
  - 🟢 Зелёный — достигнутое значение

### 3.3. Детализация по индикаторам

Основная таблица содержит:

| Столбец | Описание |
|---------|----------|
| **Код** | Уникальный код индикатора (бейдж) |
| **Индикатор** | Название и категория |
| **Ед. изм.** | Единица измерения (количество, Да/Нет, % и т.д.) |
| **Цель** | Целевое значение |
| **Достигнуто** | Фактически достигнутое значение |
| **Остаток** | Разница между целью и достижением |
| **Прогресс** | Визуальный прогресс-бар с процентом |

#### Цветовая индикация прогресса

| Прогресс | Цвет | Значение |
|----------|------|----------|
| ≥ 100% | 🟢 Зелёный | Индикатор достигнут |
| 50–99% | 🔵 Синий | Активный прогресс |
| 1–49% | 🟡 Жёлтый | Начальный этап |
| 0% | ⚪ Серый | Не начат |

#### Раскрывающаяся детализация по контрактам

При клике на индикатор раскрывается таблица с разбивкой по контрактам:

| Поле | Описание |
|------|----------|
| Контракт | Номер контракта (ссылка) |
| Подрядчик | Исполнитель |
| Цель | Целевое значение для данного контракта |
| Достигнуто | Фактическое значение |
| % | Процент выполнения (цветной бейдж) |

### 3.4. Источники данных индикаторов

Данные по индикаторам формируются из двух источников:
1. **Индикаторы контрактов** (`ContractIndicator`) — целевые значения, привязанные к контрактам
2. **Прогресс по АВР** (`ContractIndicatorProgress`) — фактические значения, вносимые через отчёты о прогрессе работ

## 4. Отчёт по KPI сотрудников

**Путь:** `/Reports/EmployeeKpi`  
**Назначение:** Анализ эффективности сотрудников по задачам и активности

![KPI сотрудников](/Users/mirqurbonnaimov/.gemini/antigravity/brain/6dcbc87c-3bb8-4347-9236-729e9a7d826d/employee_kpi_report_1771843495074.png)

### 4.1. Сводные карточки

| Карточка | Описание |
|----------|----------|
| **Сотрудников** | Общее количество активных пользователей (с указанием тех, кто имеет задачи) |
| **Всего задач** | Общее количество задач в системе (с указанием выполненных) |
| **Среднее выполнение** | Средний процент выполнения задач среди сотрудников (с прогресс-баром) |
| **Просрочено** | Количество просроченных задач, требующих внимания |

### 4.2. Графики

#### Горизонтальная столбчатая диаграмма «Задачи по сотрудникам»
- **Тип:** Стековая горизонтальная диаграмма (Stacked Horizontal Bar)
- **Ось Y:** Имена сотрудников (только с задачами)
- **Наборы данных:**
  - 🟢 Зелёный — выполненные задачи
  - 🔵 Синий — задачи в работе
  - 🔴 Красный — просроченные задачи
  - ⚪ Серый — прочие задачи

#### Кольцевая диаграмма «Статус задач»
- **Тип:** Doughnut Chart
- **Данные:** Общее распределение задач по статусам (Выполнено / В работе / Просрочено / Прочие)

### 4.3. Детализация по сотрудникам

| Столбец | Описание |
|---------|----------|
| **Сотрудник** | Аватар (инициал), ФИО и email |
| **Должность** | Должность сотрудника |
| **Всего задач** | Общее количество назначенных задач |
| **Выполнено** | Количество завершённых задач (зелёный бейдж) |
| **В работе** | Количество задач в активной работе (синий бейдж) |
| **Просрочено** | Количество просроченных задач (красный бейдж) |
| **АВР** | Количество поданных актов выполненных работ |
| **Выполнение** | Процент выполнения с прогресс-баром |

### 4.4. Источники данных

| Метрика | Источник |
|---------|----------|
| Задачи | Таблица `ProjectTasks` (по полю `AssigneeId`) |
| Статус задач | Поле `Status` (Completed, InProgress и др.) |
| Просроченные | Поле `IsOverdue` (вычисляемое) |
| АВР | Таблица `WorkProgresses` (по полю `SubmittedByUserId`) |
| Активные сотрудники | Таблица `AspNetUsers` (поле `IsActive = true`) |

### 4.5. Формула расчёта KPI

```
Процент выполнения = (Выполненные задачи ÷ Всего задач) × 100%
```

Цветовая индикация:
- 🟢 ≥ 80% — высокая эффективность
- 🔵 ≥ 50% — удовлетворительно
- 🟡 > 0% — требует улучшения
- ⚪ 0% — без задач

## 5. Управление доступом

Каждый отчёт управляется **отдельным разрешением** через систему `RoleMenuPermission`. Это позволяет гибко настраивать доступ к каждому типу отчёта для разных ролей.

### Ключи разрешений

| Отчёт | MenuKey | Описание в UI |
|-------|---------|---------------|
| Платежи | `ReportsPayments` | Отчёт: Платежи |
| Индикаторы | `ReportsIndicators` | Отчёт: Индикаторы |
| KPI сотрудников | `ReportsEmployeeKpi` | Отчёт: KPI Сотрудников |

### Настройка прав доступа

Администратор может настроить доступ к каждому отчёту через:  
**Меню → Роли → [Роль] → Разрешения**

Для каждого отчёта доступен один тип разрешения:
- ✅ **Просмотр** — доступ к отчёту

### Архитектурный паттерн загрузки

Все три отчёта используют единый паттерн асинхронной загрузки:

```
1. Пользователь открывает страницу
2. Отображается скелетон-плейсхолдер (анимация загрузки)
3. JavaScript отправляет AJAX-запрос на *Data endpoint
4. Сервер выполняет запрос к БД и возвращает Partial View
5. HTML вставляется на страницу, скелетон скрывается
6. Инициализируются Chart.js графики
```
